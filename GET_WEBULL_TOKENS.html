🔐 Webull Token Extraction Guide

⚠️ SECURITY WARNING:
Your Webull tokens = full trading account access. NEVER share them with anyone except your own bot!
📝 What You'll Extract:
  • access_token - API authentication
  • refresh_token - Renew expired access tokens
  • uuid - User identifier
  • did (Device ID) - Device fingerprint

Step 1: Open Webull in Incognito Mode

This ensures clean session without cached data interference.

  • Chrome/Edge: Press Ctrl+Shift+N (Windows) or Cmd+Shift+N (Mac)
  • Firefox: Press Ctrl+Shift+P (Windows) or Cmd+Shift+P (Mac)

Step 2: Open Developer Tools BEFORE Login

IMPORTANT: Must open DevTools BEFORE logging in to capture the tokens!

  • Press F12 or Ctrl+Shift+J (Windows) or Cmd+Option+J (Mac)
  • Click the Network tab
  • Make sure "Preserve log" checkbox is CHECKED ✅

Step 3: Login to Webull

  • Navigate to https://app.webull.com/
  • Enter your credentials
  • Complete MFA/security checks if prompted
  • Wait for page to fully load

Step 4: Find the Login Response

In the Network tab (with Preserve log enabled):

  1. In the filter box, type: login or passport
  2. Look for entries like /passport/login/v5 or /account/login
  3. Click on the request that returned status 200 (green)

Step 5: Extract Tokens from Response

  1. Click the Response tab (or Preview tab)
  2. Look for JSON containing these fields:
{ "accessToken": "dc_us1.188awefa4-xxxxx...", "refreshToken": "188a0awef2444-xxxxx...", "tokenExpireTime": "2025-12-15T13:35:35.780+0000", "uuid": "165dbaa7333aeawf312151ar1f1dc804b9b" }

Copy all four values!

Step 6: Extract Device ID (DID)

Method A: From Console (Easiest)

  1. Click the Console tab in DevTools
  2. Paste this code and press Enter:
document.cookie.match(/web_did=([^;]+)/)[1]
  1. Copy the hex string that appears (this is your DID)

Method B: From Cookies

  1. Click Application tab → Cookies → https://app.webull.com
  2. Find the web_did cookie
  3. Copy its value

Step 7: Paste into Bot Setup Wizard

When the bot asks for Webull credentials, paste:

  • Access Token: The accessToken value
  • Refresh Token: The refreshToken value
  • UUID: The uuid value
  • Device ID: The web_did cookie value
🛡️ Security Tips:
  • Tokens are stored encrypted on your computer (Windows DPAPI)
  • Access tokens expire after ~7-10 days (bot auto-refreshes using refresh token)
  • If tokens are compromised, change your Webull password immediately
  • Never commit tokens to GitHub or share publicly
❓ Troubleshooting:
  • Can't find login response: Make sure "Preserve log" is checked BEFORE logging in
  • Response is empty: Try clicking different requests with status 200
  • DID not in cookies: Make sure you're fully logged in to Webull first
  • Tokens don't work: They may have expired - extract fresh ones
