Getting Started — Onboarding Process

How BotifyTrades Works

BotifyTrades monitors Discord and Telegram channels for trade signals, parses them automatically, and executes orders on your connected broker accounts. The flow is simple:

1
Connect Discord
Link your Discord account so the bot can read signal channels
→
2

→
3
Add Channels
Tell the bot which Discord channels contain trade signals
→
4
Configure & Go
Set risk rules, sizing, and let the bot trade for you
First-Time Setup

When you first log in, the Setup Wizard will guide you through each step. You can also launch it anytime from Settings → Setup Wizard.

Recommended Setup Order

  1. Create your admin account — Set a strong password and recovery email during first login.
  2. Enter your license key — Paste your key in Settings → License or use the LICENSE_KEY environment variable.
  3. Connect Discord — Add your Discord user token (see Discord section below).
  4. Connect at least one broker — Start with paper trading to verify signals are being parsed correctly.
  5. Add your first channel — Pick a signal channel, set it to "Track" mode first to observe before going live.
  6. Configure risk management — Set stop losses and profit targets before enabling live execution.
  7. Switch to "Execute" mode — When you're confident, flip the channel to live execution.
Important

Always start with Paper Trading or Track Mode to verify that signals are parsed correctly before committing real money.

Discord Connection

What You Need

BotifyTrades uses a Discord token (your personal user token) to monitor channels you have access to. This is not a bot token from the Discord Developer Portal.

How to Get Your Discord User Token

  1. Open Discord in your web browser (not the desktop app) at discord.com/app
  2. Press F12 (or Ctrl+Shift+I) to open Developer Tools
  3. Go to the Network tab
  4. Type any message in any channel, or click on any channel
  5. In the Network tab, click on any request to discord.com
  6. In the request headers, find the Authorization header
  7. Copy the value — this is your user token
Security Warning

Your Discord token grants full access to your account. Never share it with anyone. BotifyTrades stores it encrypted in your local database.

Connecting in Settings

  1. Go to Settings page
  2. Find the Discord section
  3. Paste your token into the Discord User Token field
  4. Click Save, then Restart Bot
  5. The status badge should turn green showing "Connected"

Discord Bot Settings

Discovery Mode
When enabled, the bot logs every message it sees (across all servers and channels) to the console. Use this to find Channel IDs for channels you want to monitor. Turn it off after setup to reduce noise.
Allow Self Messages
If enabled, the bot will also process signals from your own Discord messages. Useful for testing — you can paste a signal in a channel and verify it gets parsed.
Allowed Author IDs
Comma-separated list of Discord User IDs. If set, the bot will only process signals from these specific users. Leave empty to process signals from everyone in the channel.
Allowed Guild IDs
Comma-separated list of Discord Server (Guild) IDs. If set, the bot will only listen to servers in this list. Leave empty to listen to all servers.
Signal Patterns
Regex patterns used to parse trade signals. The bot comes pre-configured with patterns that match common signal formats like BTO 5 AAPL 150C 3/21 @ 2.50. Only modify these if your signal provider uses a non-standard format.

Add Channel

→
3

→
4
Assign Broker
Pick which broker(s) execute trades from this channel
Channel Type
Execute — Live trade execution with real/paper broker. Track — Paper-only performance tracking (no real orders placed).
Channel Name
A friendly label for you to identify this channel (e.g., "VIP Alerts", "Dark Horse").
Discord Channel ID
The numeric ID of the Discord channel. Enable Discovery Mode in Settings, send any message in the channel, and look for the ID in console logs.
Select Brokers
Check one or more brokers to route this channel's signals to. E.g., check "Webull LIVE" and "Alpaca PAPER" to execute on both simultaneously.

Channel Settings Tabs

Click the gear icon on any channel card to open its settings. There are four tabs:

Execute Trades
Master toggle. When OFF, signals are detected and logged but no orders are placed. Turn this ON only after verifying your setup.
Position Size %
Percentage of your account's buying power to use per trade. E.g., 5% of $10,000 buying power = $500 per position.
Default Qty
Fixed number of contracts (options) or shares (stocks) per trade. Used when the signal doesn't specify quantity and percentage sizing isn't set.
Force My Size %
When enabled, the bot ignores the quantity from the signal and always uses your configured Position Size % for calculation.
Balance Mode
Live: Uses real-time buying power (changes throughout the day). Pre-Market: Uses a snapshot taken at 4:00 AM ET (stable baseline). Start-of-Day: Uses a snapshot taken at 9:30 AM ET market open.
Track Signals
Enables paper-trade tracking alongside live execution. Useful for comparing your performance against the signal provider.
Max Position $
Hard dollar cap on any single position. Even if % sizing calculates $2,000, this will cap it. Set to 0 to disable.
Sizing Example

Account buying power: $10,000 · Position Size: 5% · Max Position: $400

Calculated size: $10,000 × 5% = $500 → Capped at $400 (Max Position limit applies)

Exit Mode
Signal: Only exits when the signal provider posts an STC. Risk: Bot automatically exits based on your stop loss / profit target rules. Hybrid: Both active — whichever triggers first wins.
Signal Update Automation
When a signal provider edits their original Discord message (e.g., changing the stop loss), the bot detects the edit and updates your position's risk settings automatically.
SL Escalation Only
When ON, hitting profit targets does NOT sell partial positions. Instead, it only moves your stop loss up to lock in gains. Your position closes only when the trailing stop or stop loss is hit.
Slippage Protection
Prevents entering a trade if the current price has moved too far from the signal price. Protects against "chasing" old signals.
Max Slippage %
Maximum price deviation allowed. E.g., signal at $2.00 with 50% slippage = will enter up to $3.00 but reject at $3.01.
Order Chase
OFF: No order replacement. Entry Only: Chases unfilled BTO orders with better prices. Exit Only: Chases unfilled STC orders. Both: Chases both entry and exit orders. Uses mid-price replacement to get fills.
NDX → QQQ Conversion
Automatically converts expensive NDX index options ($3,000+) into equivalent QQQ options (~$300). Useful if your account can't afford NDX contracts. Target Delta controls the strike selection.
Entry Order Type
Limit: Places order at the signal price (safer, may not fill if price moves). Market: Fills immediately at current price (guaranteed fill, but may get worse price).
Trim Order Type
How partial exits (trims) are placed. Limit at the signal's trim price, or Market for immediate fills.
Stop Loss Order Type
Limit: STC at a specific price (may not fill in fast drops). Market: Guaranteed exit but at whatever the market price is. Market is recommended for stop losses.
Recommendation

Use Limit for entries and trims, and Market for stop losses. This gives you price control on entries while ensuring your stop loss always executes.

Conditional Orders
Enables "watching" orders — when a signal says "watching AAPL above $150", the bot creates a pending order that triggers when AAPL crosses $150.
Order Timeout
Auto-cancels any unfilled regular order after this many minutes. Prevents stale orders sitting in your broker account.
Conditional Timeout
Auto-cancels pending conditional orders (watching/waiting orders) after this many minutes if the trigger price is never reached.
Confirmation Buffer %
Requires the price to move an additional X% beyond the trigger before executing. Confirms momentum and avoids false breakouts.
Breakout Reset Guard
If the price is already past the trigger when the signal arrives, the bot waits for a pullback before entering. Prevents buying at the top of a move.
Trigger Offset
Adjusts the trigger price by a % or $ amount. E.g., 0.5% offset on a $150 trigger = enter at $149.25 (before) or $150.75 (after). Mode: Percent or Dollar.
Limit Cap
Caps the maximum entry price at a % above the trigger to prevent chasing extended moves. E.g., 5% cap on $150 trigger = max entry at $157.50.

Ticker Filter

Controls which tickers a channel can trade. Found on each channel's settings panel (funnel icon).

Filter Mode
Off: Trade all tickers from this channel. Allow List: Only trade tickers you specify. Block List: Trade everything except tickers you specify.
Ticker List
Comma-separated list of tickers. For options, the underlying symbol is matched (e.g., entering "SPY" blocks all SPY options). Case-insensitive.
Example

A signal provider is great at SPY and QQQ calls but loses money on individual stocks.

Filter Mode: Allow List · Tickers: SPY, QQQ

Now only SPY and QQQ signals from this channel will be executed. All other tickers (AAPL, TSLA, etc.) are silently skipped.

Trade Summary

Found in the Advanced tab of the risk management panel. When enabled, the bot posts a summary message to the signal's Discord channel whenever a position is closed (STC).

Trade Summary
When ON, posts a formatted P&L summary embed to the Discord channel showing entry/exit price, quantity, profit/loss amount, and percentage. Useful for tracking performance and sharing results.

Trim & Stop Loss Order Types

Controls how the bot places exit orders when profit targets or stop losses trigger. Found in the Targets & SL tab of the risk management panel.

Trim Order Type
Market: Sells immediately at whatever the market price is. Guarantees a fill but may get a slightly worse price. Limit: Places the sell at a specific price with an offset you configure.
Trim Limit Offset
When Trim Order Type is Limit, this controls how far below the target price the limit order is placed. Choose Dollar (e.g., $0.05 below) or Percent (e.g., 2% below). A small offset helps ensure the limit order fills while still getting close to the target price.
Stop Loss Order Type
Limit: Places a limit sell at the SL price plus an offset. May not fill in a fast crash. Market: Guarantees exit but at whatever price is available. Market is recommended for stop losses to ensure you always get out.
SL Limit Offset %
When SL Order Type is Limit, the limit price is set this % below the stop loss trigger price. E.g., SL triggers at $9.00 with 3% offset = limit sell at $8.73. Wider offset increases fill probability in fast drops but accepts a worse price.
Example

Entry at $10.00, Stop Loss at 10% ($9.00), SL Order Type: Limit, Offset: 3%

When price drops to $9.00, the bot places a limit sell at $8.73 (3% below $9.00). If the price is between $8.73 and $9.00, the order fills. If price crashes below $8.73 instantly, the limit order may not fill.

For fast-moving options, use Market SL orders to guarantee exits.

Risk Management

How Risk Management Works

The risk engine continuously monitors all your open positions and automatically exits them based on your configured rules. It checks positions every few seconds using real-time streaming prices.

BTO signal parsed and order placed
→
2
Position Opened
Order fills, risk engine starts monitoring
→
3
Price Monitored
Checks SL, PT, trailing stop every cycle
→
4
Auto Exit
Closes position when any rule triggers

Global Risk Settings (Settings Page)

These settings apply as defaults to all channels. Individual channels can override them.

Enable Monitoring
Master toggle for the risk engine. When OFF, no automatic exits occur — positions are only closed when signal providers post STC messages.
Monitoring Interval
How frequently (in seconds) the risk engine checks positions. Lower = faster response but more CPU. Default: 30s. Minimum: 0.2s for aggressive monitoring.
Exit Strategy Mode
Signal: Only exits on provider STC signals. Risk: Only uses automated SL/PT/trailing. Hybrid: Both active, first trigger wins. Hybrid is recommended.

Stop Loss

Automatically sells your position if it drops below a certain percentage from your entry price, limiting your maximum loss.

Stop Loss Example: Entry at $10.00, Stop Loss at 10%

Example

You buy AAPL at $10.00 with a 10% Stop Loss.

If AAPL drops to $9.00 (10% below entry), the bot automatically sells your entire position.

Maximum loss: $1.00 per share (10% of entry price).

Profit Targets (Up to 4 Tiers)

Automatically sells a portion of your position at predetermined profit levels, locking in gains progressively.

4-Tier Profit Target Example: Entry at $10.00
PT1: +10%
PT2: +25%
PT3: +50%
PT4: +100%
Entry $10.00 $11.00 $12.50 $15.00 $20.00
Example with 10 contracts at $10.00

PT1 (+10% = $11.00): Sell 25% → Sell 2 contracts, keep 8

PT2 (+25% = $12.50): Sell 25% → Sell 2 contracts, keep 6

PT3 (+50% = $15.00): Sell 25% → Sell 2 contracts, keep 4

PT4 (+100% = $20.00): Sell remaining 25% → Sell 2 contracts, keep 2 runners

Result: Locked in profits at each level while keeping "runners" for further upside.

Custom Trim Quantities

By default, the bot auto-calculates how many contracts to sell at each tier (divides evenly). You can override this by specifying exact quantities for P1-P4 Qty. Leave empty for auto-calculation.

Example: 10 contracts with P1 Qty=1, P2 Qty=2, P3 Qty=3, P4 Qty=4 — sells 1 at PT1, 2 at PT2, 3 at PT3, 4 at PT4. If the total exceeds your position, the bot caps at available quantity.

Custom Trim Percentages

Instead of exact quantities, you can specify what percentage of the sellable position to trim at each tier. Percentages don't need to total 100%. Leave empty for auto-calculation.

Priority: Custom Qty > Custom Trim % > Auto-split. If both qty and trim % are set for the same tier, the qty takes precedence.

Example: 10 contracts, Runner 20%, P1 Trim=80%, P2 Trim=20% — sellable is 8 (2 runner). Sells floor(8*0.80)=6 at PT1, floor(8*0.20)=1 at PT2, 1 remaining auto-allocated.

Trailing Stop

A dynamic stop that follows the price up. Once your position gains a certain amount, the trailing stop activates and moves with the price, never moving back down.

Trailing Stop Example: 5% trail, activates after 10% gain
Entry $10.00
Price rises to $12.00
Trail: $11.40 (5% below peak)
Peak $14.00
Trail: $13.30
Drops to $13.30
SOLD at $13.30
Trailing Stop %
How far below the highest price the stop is placed. 5% trail means the stop always sits 5% below the peak. Lower = tighter (exits sooner), Higher = more room.
Activation Threshold
The minimum gain required before the trailing stop activates. E.g., 10% means the trail only starts after you're up 10%. Before that, only the fixed stop loss protects you.

Daily P&L Limits

Protects you from catastrophic days by automatically pausing trading when daily loss or profit thresholds are reached.

Max Daily Loss ($)
If your account drops by this dollar amount from the start-of-day snapshot, all new BTO orders are blocked. Open positions continue to be monitored.
Max Daily Loss (%)
Same as above but as a percentage of your starting balance. E.g., 3% of $10,000 = $300 loss limit.
Max Daily Profit ($)
Locks in gains by stopping new trades after hitting a profit target. Prevents giving back a great day's gains.
Warning Threshold (%)
Sends a notification when you're approaching your daily limit. E.g., 80% warning = alerts at $240 loss if your limit is $300.
Reset Time
Market Open: Resets at 9:30 AM ET. Pre-Market: Resets at 4:00 AM ET. Midnight: Resets at 12:00 AM ET.

Max Profit Giveback Guard

Prevents giving back too much unrealized profit. If your position reaches a high watermark and then retraces by a set percentage, it exits to protect those gains.

Example

Giveback Guard: 30%. Position peak unrealized profit: $500.

If profit drops from $500 to $350 (gave back 30% of $500), the bot exits to lock in $350 profit.

Without this guard, the position might retrace all the way back to breakeven or loss.

Circuit Breaker

An emergency shutoff that pauses all trading after a series of consecutive losses. Prevents emotional trading and cascading losses.

Circuit Breaker
When enabled, if you hit a configured number of consecutive losing trades, all new entries are blocked until you manually reset or the daily reset time passes.
Max Open Positions
Hard limit on how many positions can be open simultaneously across all channels and brokers. New signals are rejected when this limit is reached.

Exit Strategy Modes Explained

The Exit Strategy Mode controls which exit mechanisms are active for a position. This is the most important setting to understand.

📨
Signal Mode
Only exits when the signal provider posts a sell command (STC). No automated stop loss, no profit targets, no trailing stops. If the provider enables EMA risk for the channel, EMA exits will still run independently.
Least protection — fully depends on the signal provider
🤖

The bot manages all exits automatically using your configured rules: Stop Loss, Profit Targets, Dynamic SL, Early Trailing, Giveback Guard, and Legacy Trailing. Signal provider STC messages are ignored.
Full automated protection
⚡

Both active — whichever triggers first wins. If the signal provider posts STC before your stop loss hits, the position closes. If your PT1 hits before the provider trims, the bot trims automatically.
Recommended — best of both worlds

Early Trailing Stop

A smarter trailing stop that locks in breakeven as soon as a small profit is reached, then steps up your stop in fixed increments as the price climbs. Much faster than the legacy trailing stop.

Activation %
The minimum profit needed before the early trailing stop activates. When hit, your stop immediately moves to breakeven (entry price). Example: 5% activation means once you're up 5%, your stop moves to your entry price — you can't lose money anymore.
Step %
After activation, for every additional X% of profit above the activation level, the stop ratchets up by that same X%. The stop never moves down — it only steps higher.
Early Trailing Example: Entry $10.00, Activation 5%, Step 3%
Price hits $10.50
+5% → Stop moves to $10.00 (breakeven)
Price hits $10.80
+8% → Step 1 → Stop at $10.30 (+3%)
Price hits $11.10
+11% → Step 2 → Stop at $10.60 (+6%)
Price hits $11.40
+14% → Step 3 → Stop at $10.90 (+9%)
Price drops to $10.90
SOLD at $10.90 — +9% profit locked
How Step Calculation Works:
Entry: $10.00  |  Activation: 5%  |  Step: 3%

Current P&L: +11%
Steps earned: floor((11% - 5%) / 3%) = 2 steps
New stop at: 2 steps × 3% = +6% → $10.00 × 1.06 = $10.60

The stop only moves up — if price drops from +11% to +9%, the stop stays at $10.60
Early Trailing vs. Legacy Trailing

These two are mutually exclusive. If Early Trailing is enabled, Legacy Trailing is automatically skipped. Early Trailing is recommended for most strategies because it locks breakeven much faster.

Dynamic Stop Loss Profiles

Dynamic SL automatically moves your stop loss up each time a profit target is hit. There are 3 built-in profiles that control how aggressively the stop escalates.

After PT Hit Conservative Standard Aggressive
PT1 Hit SL → Entry (breakeven) SL → Entry (breakeven) SL → Entry - 2%
PT2 Hit SL → Entry + 3% SL → Entry + 5% SL → Entry (breakeven)
PT3 Hit SL → Entry + 8% SL → Entry + 10% SL → Entry + 8%
PT4 Hit SL → Entry + 15% SL → Entry + 17% SL → Entry + 15%
Conservative
Keeps stops close to entry until deep profits are reached. PT1 moves SL to breakeven, PT2 to +3%, PT3 to +8%, PT4 to +15%. Best for volatile assets that often retrace before continuing higher.
Standard
Balanced approach. PT1 locks breakeven, PT2 moves SL to +5%, PT3 to +10%, PT4 to +17%. Recommended for most traders.
Aggressive
Allows a small loss even after PT1 hit (-2% below entry). PT2 moves SL to breakeven, PT3 to +8%, PT4 to +15%. Best for scalping strategies where you want the most room for momentum to build.
Standard Profile Example: Entry at $10.00

PT1 hits (+10% = $11.00) → SL moves from $9.00 to $10.00 (breakeven locked)

PT2 hits (+25% = $12.50) → SL moves up to $10.50 (+5% guaranteed)

PT3 hits (+50% = $15.00) → SL moves up to $11.00 (+10% guaranteed)

Price reverses and drops to $11.00 → Dynamic SL triggers → EXIT with +10% profit

Without Dynamic SL, the position might have dropped all the way back to the original $9.00 stop loss = -10% loss.

Dynamic SL requires Profit Targets

Dynamic SL only activates after at least one profit target is hit. If you have no profit targets configured, the Dynamic SL will never fire. Make sure to set PT1-PT4 alongside Dynamic SL.

Leave Runner

Reserves a percentage of your original position that is never sold by profit targets. The runner rides the trade for maximum upside and is only closed by a stop loss, trailing stop, or manual exit.

Master toggle. When ON, a percentage of your position is reserved and excluded from profit target trims.
Runner %
The percentage of your original position to keep as a runner. E.g., 20% of 10 contracts = 2 contracts held back from all PT trims.

Giveback Guard — In Detail

Tracks the highest unrealized profit your position has ever reached. If the position gives back too much of that peak profit, it forces an exit to protect your gains.

Giveback Guard
Master toggle. When ON, the guard tracks your position's high-water mark and exits if too much profit is given back.
Allowed Giveback %
The maximum percentage of peak profit you're willing to lose. E.g., 30% means if your peak was +$500 and it drops to +$350 (gave back 30%), the guard triggers.

When does the Giveback Guard activate?

The guard arms itself when PT2 is hit OR when the max P&L exceeds the early trailing activation threshold (or 30% if no trailing is set). Before activation, it does nothing — this prevents premature exits on normal early-trade volatility.

EMA Risk Management

Uses Exponential Moving Average (EMA) analysis on candle data to detect when a trade's momentum has reversed. This is a trend-based exit that fires independently of price-based stops.

EMA Risk
Master toggle. When ON, the bot monitors the EMA cross state (price crossing above or below the EMA line) and exits when momentum fades.
EMA Period
Number of candles used to calculate the EMA. Lower = more sensitive (faster exits). Default: 5 candles. Options: 3 (Fast), 5 (Standard), 8 (Moderate), 13 (Slow), 21 (Very Slow).
Candle Timeframe
Candle size used for EMA calculation. Options: 1-minute, 2-minute, 3-minute, 5-minute. Smaller timeframes react faster to reversals. Default: 5 minutes.
Buffer %
A small buffer zone around the EMA line. Price must cross the EMA by at least this percentage before an exit is considered. Prevents false triggers from tiny EMA touches. Default: 0.1%. Higher values (e.g., 0.5%) give more room but delay exits.
Exit on Cross
When ON, the bot exits the position (full or partial if Leave Runner is enabled) when price closes a candle below the EMA line (plus buffer). This is the primary EMA exit trigger. When OFF, EMA data is still collected but no exit action is taken.
Stop Escalation
Instead of exiting immediately on an EMA cross, the bot first tightens the stop loss to near the EMA level (with a small buffer). Only exits if that tightened stop is then hit. Gives the trade one more chance to recover above the EMA.
No-Trend Candles
Number of consecutive candles that must close below the EMA before an exit triggers. Default: 3. Higher values require stronger confirmation of trend reversal but delay the exit. Set to 1 for fastest response.
Use Underlying Chart
For options positions, uses the underlying stock's price chart (e.g., SPY) instead of the option contract price to calculate the EMA. Recommended ON because stock prices are smoother and more reliable for trend detection than volatile option premiums.
Extended Hours
When ON, includes pre-market (4:00 AM) and after-hours (until 8:00 PM) candle data in EMA calculations. When OFF, only regular market hours (9:30 AM - 4:00 PM ET) are used. Enable this if your strategy trades during extended hours.

EMA has a warmup period

The EMA evaluator requires 2 candles after entry before it can make any decisions, plus enough candle history to calculate the EMA itself (default 5 candles). During the warmup period, EMA exits are frozen and will not fire.

Signal Mode + EMA

In Signal exit strategy mode, EMA is the only automated exit that runs. All other risk settings (SL, PT, trailing) are stripped. EMA exits act independently in this mode, giving signal-followers an additional safety net.

SL Escalation Only Mode

A special mode where hitting profit targets does NOT sell any shares. Instead, each PT hit only moves the stop loss higher. Your full position stays open until the escalated stop loss is finally hit.

When to use Escalation Only

Use this when you want to ride the full position for maximum profit and only exit when the trend reverses. The escalating stop loss still protects you — after hitting PT3 with Standard profile, your stop is at +10%, so even in a crash you keep +10% on your entire position.

Each Setting Explained

📈
Entry Confirmation (%)
Channels → Conditional Tab → Entry Confirmation
What it does: Converts a normal buy signal into a conditional (watched) order. The bot will NOT buy immediately — it monitors the price and only buys when the stock moves up by this percentage above the signal price.

Why use it: Filters out fake breakouts. If a signal says “GXAI over $2.60” but the price never reaches $2.60, no money is spent.
Signal price: $2.60
Confirmation: 2%
Formula: $2.60 × (1 + 2/100) = Trigger @ $2.652
Bot waits until price ≥ $2.652 before buying
🎯
Trigger Offset (% or $)
Channels → Conditional Tab → Trigger Offset
What it does: Fine-tunes the trigger price with an additional adjustment. Positive values push the trigger further away (more conservative). Negative values bring it closer (enters sooner).

When it applies: Applied after Entry Confirmation at conditional order creation.
Trigger (after confirmation): $2.652
Offset: -1% (enter sooner)
Formula: $2.652 × (1 - 1/100) = Adjusted trigger @ $2.625
Dollar mode example:
Trigger: $2.60    Offset: +$0.10
Formula: $2.60 + $0.10 = Adjusted trigger @ $2.70
🔄
Breakout Reset Guard
Channels → Conditional Tab → Breakout Reset
What it does: If the price is already past the trigger when the conditional order is created, this guard waits for the price to pull back below the trigger first, then watches for the real breakout. Prevents immediate execution on stale signals.

Default: ON. Disable if you want immediate trigger on already-passed prices.
Trigger: $2.60    Current price: $2.75 (already past!)
Guard ON: Wait for pullback below $2.60, then trigger on next cross above
Guard OFF: Trigger immediately since $2.75 ≥ $2.60
🛡️
Limit Cap (% above trigger)
Channels → Execution Tab → Price Controls → Limit Cap
What it does: Sets a maximum price you’re willing to pay. When the conditional order triggers, instead of chasing the stock at any price, the bot places a limit order capped at trigger + this percentage. If the stock has already run past your cap, the order simply won’t fill — protecting you from overpaying.

Important: Limit Cap acts as a hard ceiling even if the channel is set to “Market” order mode. It always overrides market mode when enabled.
Trigger price: $2.60
Limit Cap: 5%
Formula: $2.60 × (1 + 5/100) = Max buy price: $2.73
Order won’t fill above $2.73 regardless of market price
📥

Channels → Order Types Tab → Entry Order
What it does: Controls whether the bot places limit orders (at a specific price) or market orders (at whatever the current price is) when entering a trade.

⏱
Conditional / Order Timeout
Channels → Conditional Tab → Timeouts
What it does: Auto-cancels the conditional order if the trigger is never reached within the timeout period. “Order Timeout” applies to all orders. “Conditional Timeout” applies only to conditional orders (overrides the general one if set).
Example: Conditional timeout = 60 minutes
Signal at 10:00 AM → Order auto-cancels at 11:00 AM if trigger not reached
🏃
Slippage Protection
Channels → Execution Tab → Slippage Protection
What it does: After a conditional order triggers, checks the current market price against the trigger price. If the stock has moved more than your max slippage %, the bot waits (up to the configured minutes) for the price to come back before placing the order.

Price at trigger: $3.50 (+34.6%) → Within 50%, proceeds with order
Price at trigger: $4.20 (+61.5%) → Waits up to 5 min for pullback
