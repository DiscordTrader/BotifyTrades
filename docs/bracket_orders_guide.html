What Are Bracket Orders?

A bracket order is a pair of exit orders placed on the broker immediately after your entry fills:


Stop Loss (SL) A STOP order at a price below entry. If the stock drops to this price, broker auto-sells to limit your loss.
Profit Target (PT) A LIMIT order at a price above entry. If the stock rises to this price, broker auto-sells to lock in profit.

💡
Why does this matter? Broker-side orders execute even if BotifyTrades is offline or your internet drops. Without bracket orders, the risk engine manages exits in software — which requires the bot to be running.

The 4 Bracket Modes

Each channel can independently choose how bracket orders are handled via broker_bracket_mode in the channel's risk settings.

🛡
Both
both
SL and PT placed on broker.
Maximum protection — broker handles both exits even if bot goes offline.
🛑
SL Only
sl_only
Only Stop Loss on broker.
PT managed by risk engine in software. Useful for Webull options that reject simultaneous orders.
🎯
PT Only
pt_only
Only Profit Target on broker.
SL managed by risk engine. Useful when you want dynamic trailing stops without broker interference.
⚪
Disabled
none
No broker orders for SL/PT.
Risk engine manages all exits internally. Bot must be running to exit positions.
⚠️
Default is both — this is the safest option. Only change to sl_only, pt_only, or none if you understand the trade-offs. With none, the bot must be online to manage exits.

SCHWAB Native OCO + TRIGGER

Schwab is the only broker with full native bracket support. It uses a TRIGGER parent order wrapping an OCO (One-Cancels-Other) child group.

🟢 Bracket ENABLED (mode: both)

After entry fills, the risk engine places two independent orders on Schwab:

Entry fills at $100.00 (100 shares) │ ├── STOP ORDER (SL) │ ├─ schwab.place_stop_order() │ ├─ symbol: AAPL, qty: 100 (full position) │ ├─ stop_price: $95.00 (5% below entry) │ ├─ session: NORMAL (forced — SEAMLESS rejects STOP) │ └─ duration: GOOD_TILL_CANCEL │ └── LIMIT ORDER (PT1) ├─ schwab.place_stock_order(action='STC', price=$110) ├─ qty: 40 (PT1 trim — e.g. 40% of position) ├─ price: $110.00 (10% above entry) └�� _skip_cancel_check=True ← prevents SL from being cancelled!
⚠️
Critical: _skip_cancel_check=True
Without this flag, Schwab's _cancel_conflicting_sell_orders() would detect the SL stop as a "conflicting sell order" and cancel it when placing the PT limit. This was the root cause of the SL cancellation bug fixed in v9.3.0.

WEBULL Simulated Brackets

Webull does NOT support native OCO or bracket orders. BotifyTrades simulates brackets by placing 3 separate, unlinked orders.

🟢 Bracket ENABLED (mode: both)

Entry fills at $100.00 │ ├── Order 1: ENTRY │ └─ webull.place_stock_order('BTO') — MKT or LMT, GTC │ ├── Order 2: STOP LOSS (independent — NOT linked to entry) │ └─ webull.place_stock_order('STC') — STP type │ └── Order 3: PROFIT TARGET (independent — NOT linked) └─ webull.place_stock_order('STC') — LMT type ⚠ WARNING: These are 3 SEPARATE orders! If PT fills, the SL is NOT auto-cancelled. Risk engine must manually cancel the remaining order.
⚠️
Webull Limitation: Since orders aren't linked, if PT fills, the risk engine must detect the fill and cancel the SL order. If the bot is offline when PT fills, the SL remains active and could trigger an unwanted sell on a subsequent dip. The sl_only mode is recommended for Webull options.

IBKR Risk Engine Only

Interactive Brokers wrapper has no bracket, OCO, or stop order methods implemented. All exit management is handled entirely by the risk engine.

Progressive PT Tier Cascade

📈 Multi-Tier Profit Taking

BotifyTrades supports up to 4 profit target tiers. When each tier fills, the next one is automatically placed on the broker (if bracket mode allows PT).

SL
Stop Loss — Full position qty (e.g., 100 shares at $95)
Placed immediately after entry fill. Stays active until position closes.
1
PT1 — Trim qty (e.g., 40 shares at $110)
Placed with entry. When filled → cancel PT1 → place PT2 → resize SL qty.
2
PT2 — Second trim (e.g., 30 shares at $115)
Auto-placed when PT1 fills. When filled → cancel PT2 → place PT3 → resize SL qty.
3
PT3 / Runner — Remaining qty (e.g., 30 shares at $120)
Final tier. Can be a "runner" left for maximum gains with trailing stop.

🔄 Cascade Flow (Schwab Example)

PT1 fills (40 shares sold at $110) │ ├── Cancel old SL order (was 100 shares) ├── Place new SL order (now 60 shares — remaining qty) └── Place PT2 LIMIT order ├─ qty: 30, price: $115 └─ _skip_cancel_check=True PT2 fills (30 shares sold at $115) │ ├── Cancel old SL order (was 60 shares) ├── Place new SL order (now 30 shares) └── Place PT3 LIMIT order ├─ qty: 30, price: $120 └─ _skip_cancel_check=True Each tier: cancel prev PT → cancel old SL → place new SL (resized) → place next PT

Session & Extended Hours

🕐 Schwab Session Types

Session Hours Supports STOP? Used For
NORMAL 9:30 AM – 4:00 PM ET ✓ Yes All order types including STOP
SEAMLESS Pre-market + Regular + After-hours ✗ No Extended hours LIMIT and MARKET only

⚠️
Schwab rejects STOP orders in SEAMLESS session.
The broker automatically forces session = "NORMAL" for all stop orders (place_stop_order line 2609) and OCO stop legs (place_oco_order line 2901). This means bracket SL orders only execute during regular market hours.
