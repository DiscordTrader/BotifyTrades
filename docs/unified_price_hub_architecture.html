What is the Unified Price Hub?

The UnifiedPriceHub (UPH) is a singleton, thread-safe price aggregation engine that collects real-time prices from all connected broker streaming hubs (Webull MQTT, Schwab WebSocket, IBKR TWS, Tastytrade DXLink, Trading212) into one centralized cache. It serves as the single source of truth for all price consumers in the system.

Previously operating in Shadow Mode (observe-only), the UPH is now promoted to Active Mode — it directly supplies prices to the Risk Engine, Conditional Orders, and Streaming Price Monitor with built-in frozen detection and automatic cross-broker recovery.

Key Design Principles

Zero-Latency Streaming Path

Hub events fire to consumers via on('quote_updated') callbacks. UPH subscribes as a parallel listener — it does NOT sit in the middleware path. Existing broker → consumer latency is unchanged (sub-millisecond).

Event + Poll Dual Mode

Real-time events provide instant tick-by-tick data. The 1-second poll loop catches any missed events via poll_all_hubs() and runs the frozen detection sweep.

Graceful Degradation

If UPH is not running or returns no quote, all three consumers fall back to their original price pipelines. UPH is additive — never subtractive.

Thread-Safe Singleton

Double-checked locking via _create_lock. All mutable state protected by dedicated locks. Lock ordering enforced to prevent deadlocks.

Symbol Canonicalization

Index symbols (SPXW→SPX, NDXP→NDX, VIXW→VIX) are canonicalized on entry. Per-hub symbol maps translate back for subscription and lookup.

Dynamic Freshness

Freshness is recomputed on every get_quote() call using real-time price_age_seconds. Consumers always see current freshness, never stale classifications.

Price Data Flow — From Broker to Consumer

There are two pathways for price data to reach the UPH cache, plus one recovery pathway:

Path 1: Real-Time Event Subscription (Primary — Millisecond Latency)

1

Broker Hub emits quote_updated event

Each broker data hub (WebullDataHub, SchwabDataHub, etc.) fires an event when a new price tick arrives from their streaming connection (MQTT, WebSocket, TWS API, DXLink).

2

UPH callback _on_quote_updated() fires

During _subscribe_to_hubs(), UPH registers hub.on('quote_updated', callback) for each hub. The callback extracts bid/ask/last/greeks from the event data. UPH is a parallel subscriber — Risk Engine and other consumers also subscribe directly.

3

_update_cache() stores in unified cache

Canonicalizes symbol, merges with existing quote, detects price changes (last_changed_ts), classifies freshness, emits quote_updated event. Rejects stale data if incoming timestamp < existing - 1 second.

// Event subscription chain hub.on('quote_updated', callback) → parallel to Risk Engine's own subscription → callback extracts {bid, ask, last, volume, greeks, timestamp} → _update_cache(symbol, data, 'webull') → canonicalize: SPXW → SPX → merge into existing UnifiedQuote → track last_changed_ts (price movement detection) → classify freshness: fresh/aging/stale/degraded/unverified → emit('quote_updated', {...})

Path 2: Poll-Based Bulk Sync (Backup — 1 Second Interval)

1

Poll loop calls poll_all_hubs() every 1 second

Iterates all connected hubs that have is_streaming() == True and get_all_quotes(). Bulk-reads entire quote cache from each hub.

2

Each quote is fed to _update_cache()

Same merge logic as event path. Stale-data rejection prevents poll from overwriting fresher event data. Acts as a safety net to catch any events that were missed.

Path 3: Recovery — Cross-Broker Probe + REST Fallback

1

Frozen sweep detects stale price

_frozen_sweep() runs every 1s. If a monitored symbol's price_age_seconds exceeds threshold (2s regular / 15s extended hours), it's flagged as frozen.

2

_trigger_cross_broker_probe()

Checks ALL other broker hubs (excluding the frozen symbol's source hub) for a fresh price within 3 seconds. If found, updates cache and clears frozen flag.

3

_try_rest_probe() — Last resort

If no cross-broker has fresh data, falls back to REST API calls (Schwab or Webull). Rate-limited to 1 call per symbol per 2 seconds. Uses RateLimitManager to rank brokers by remaining API budget.

Consumer Read Path

// When Risk Engine / Conditional Orders need a price: uph.get_quote(symbol) → _to_canonical(symbol) // SPXW → SPX → cache lookup (O(1) dict get) → if found: recompute freshness // dynamic, not stored stale value return UnifiedQuote → if miss: _try_fill_from_hubs() // query all live hubs directly → if still miss: return None // consumer falls back to own pipeline
Key: Freshness is recomputed dynamically in get_quote() before returning. When consumers check quote.freshness in ('fresh', 'aging'), they see the current age — not the age at cache-write time. This prevents stale freshness classification from allowing old prices through.

Frozen Price Detection Engine

The frozen detection sweep runs every 1 second inside the poll loop. It monitors all symbols registered by the Risk Engine and conditional orders, detecting when streaming prices stop updating.

How Frozen Detection Works

1

Market Hours Gate

_is_market_open() checks if we're in US/Eastern 4:00 AM - 8:00 PM weekdays. Frozen detection is disabled outside market hours — no false positives overnight.

2

Threshold Selection

Regular hours (9:30 AM - 4:00 PM ET): 2 seconds. Extended hours: 15 seconds. Extended hours have lower tick frequency, so a longer threshold avoids false positives.

3

Snapshot Monitored Symbols

Takes a thread-safe snapshot of _monitored_symbols. These are symbols registered by the Risk Engine's _update_monitored_symbols() — all open position equity underlyings.

4

Check Each Cached Symbol

For every monitored symbol in cache: compute price_age_seconds (current time minus last_changed_ts). If age exceeds threshold → newly frozen. If age is below threshold and was frozen → recovered.

5

Grace Period for Uncached Symbols

Symbols in _monitored_symbols but NOT in cache get a 5-second grace period from when monitoring started. This prevents a flood of false frozen alerts at startup before streaming data arrives.

6

Trigger Recovery & Re-probe

Newly frozen symbols immediately trigger _trigger_cross_broker_probe(). Already-frozen symbols are re-probed every 2 seconds (cooldown).

Frozen Sweep Code Flow

_frozen_sweep() — runs every 1 second in poll loop │ ├── if not _is_market_open() → return (skip outside market hours) │ ├── threshold = 2.0s (regular) or 15.0s (extended) │ ├── snapshot monitored_symbols set (thread-safe copy) ├── snapshot cache entries for monitored symbols only │ ├── for each cached monitored symbol: │ ├── if price_age >= threshold → mark frozen, trigger probe │ └── if price_age < threshold and was frozen → mark recovered │ ├── if grace period elapsed (5s): │ └── for uncached monitored symbols → mark frozen, trigger probe │ └── for already-frozen symbols past probe cooldown (2s): └── re-trigger cross-broker probe

Freshness Classification

Classification Age Threshold Meaning Consumer Action
fresh 0 - 3 seconds Price recently changed, highly reliable Use directly ✓
aging 3 - 5 seconds Price hasn't changed in a few seconds, still acceptable Use directly ✓
stale 5 - 10 seconds Price is getting old, may be frozen Rejected by consumers
degraded 10 - 30 seconds Price is significantly old Rejected by consumers
unverified > 30 seconds Price is unreliable Rejected by consumers
All three consumers (Risk Engine, Conditional Orders, StreamingPriceMonitor) only accept quotes where freshness in ('fresh', 'aging') — effectively a 5-second price validity window.

Cross-Broker Recovery Pipeline

When a frozen price is detected, recovery follows a tiered escalation strategy designed to minimize latency and API usage.

Tier 1: Cross-Broker Streaming Probe

Fastest Recovery — Sub-Second

Checks all OTHER broker hub caches for a fresh price. This costs zero API calls — it reads from data already received via streaming.

_trigger_cross_broker_probe(symbol) │ ├── Get current quote's source_hub (e.g., "webull") │ ├── for each hub except source_hub: │ ├── if not hub.is_streaming() → skip │ ├── if not _is_hub_live(hub) → skip (no quote in 120s = dead) │ │ │ ├── lookup = _symbol_for_hub(symbol, hub_key) // SPX→SPXW for Schwab │ ├── price = _get_hub_price_if_fresh(hub, lookup, max_age=3s) │ │ │ ├── if price found AND different: │ │ ├── _update_cache() with new price │ │ ├── clear frozen flag │ │ └── return ✓ │ │ │ └── if price found AND same (confirmed fresh): │ ├── clear frozen flag (price is just not moving) │ └── return ✓ │ └── No cross-broker found → fall through to Tier 2
Same-price handling: If another broker confirms the same price with a fresh timestamp, the symbol is unfrozen. This prevents symbols from being stuck as "frozen" when the underlying price is legitimately not moving (e.g., slow-moving stocks during lunch hours).

Tier 2: Smart REST Fallback

API-Budget-Aware — Rate Limited

Only triggered when ALL broker streaming caches fail. Uses the system's RateLimitManager to choose the broker with the most remaining API budget.

_try_rest_probe(symbol) │ ├── Rate limit check: 2s cooldown per symbol │ ├── _rank_rest_brokers(rate_limit_manager) │ ├── Candidates: ['schwab', 'webull'] only │ ├── Skip any broker in backoff │ └── Sort by remaining window budget (highest first) │ ├── for each ranked broker: │ ├── Check rate_limit_manager.can_make_request() │ │ │ ├── if schwab: │ │ ├── Get access_token from schwab_auth.get_token_manager() │ │ ├── GET https://api.schwabapi.com/marketdata/v1/quotes │ │ ├── timeout=5s │ │ └── Parse bid/ask → mid, or lastPrice if regular hours │ │ │ ├── if webull: │ │ ├── Lazy-init webull REST client │ │ ├── client.get_quote(stock=symbol) with 5s thread timeout │ │ ├── Parse bidPrice/askPrice → mid │ │ ├── Fallback: pPrice (premarket) during extended hours │ │ └── Fallback: last only during regular hours │ │ │ └── if price > 0: │ ├── record_request() in rate_limit_manager │ ├── _update_cache() with source='rest/schwab' or 'rest/webull' │ ├── clear frozen flag │ └── return ✓

Proactive Subscription Manager

When the Risk Engine opens a position, it calls uph.register_monitored_symbols(). The UPH doesn't wait for a freeze to subscribe on other brokers — it proactively subscribes the symbol on ALL available streaming hubs at position-open time.

Schwab

hub.request_subscribe_equities(symbols)
Batched subscription through WebSocket pending queue. Drain interval reduced to 3s.

IBKR

hub.subscribe_symbol(sym)
Individual subscription via TWS API. One-by-one to respect IBKR's rate limits.

Tastytrade

hub.subscribe_symbol(sym)
Individual subscription via DXLink. Same pattern as IBKR.

Symbol Canonicalization: All symbols are canonicalized via _to_canonical() before storing in _monitored_symbols. This ensures SPXW, SPX all resolve to the same canonical key (SPX), preventing missed detections for index symbols.

Three Independent Price Pipelines

The system has three independent consumers that need real-time prices. Each one now tries the UPH first, with their original pipeline as fallback.

Pipeline 1: Risk Engine — position_monitor.py

Purpose: P&L calculation, stop-loss triggers, portfolio risk assessment

The Risk Engine updates all open position prices every cycle. It has the most complex price pipeline with multiple sources and a REST repair guard.

_update_prices_from_hub(positions) │ ├── Step 1: Direct hub prices (existing — unchanged) │ └── Each position's broker hub → get_quote_price() │ ├── Step 2: Cross-broker hub (existing — unchanged) │ └── For unfilled positions, try other broker hubs │ ├── Step 3: UPH fill (NEW) │ ├── for each unfilled position: │ │ ├── uph.get_quote(symbol) │ │ ├── if freshness in ('fresh', 'aging'): │ │ │ ├── Option sanity check (reject if price/avg_cost > 50x) │ │ │ ├── if REST repair active: │ │ │ │ └── Only override if UPH price differs significantly │ │ │ └── else: use UPH price directly │ │ └── else: skip (stale/degraded/unverified) │ └── Log "UPH(N)" count in price update summary │ ├── Step 4: REST fallback (existing — unchanged) │ └── Webull REST for remaining unfilled positions │ └── total = hub + cross_broker + uph + rest

REST Repair Guard

When the Risk Engine makes a REST API call and gets a fresh price, it creates a "repair" entry that blocks hub/UPH prices for 30 seconds — unless the new price differs significantly (>0.5% or >$0.02). This prevents a known-good REST price from being overwritten by a potentially frozen hub price.

Monitored Symbols Registration

_update_monitored_symbols(positions) — called by Risk Engine │ ├── Collect all equity symbols from open positions ├── uph.register_monitored_symbols(equity_symbols) │ ├── Canonicalize each symbol │ ├── Add to _monitored_symbols set │ └── Proactively subscribe on Schwab, IBKR, Tastytrade │ └── This is what populates the frozen sweep's watchlist

Pipeline 2: Conditional Order Service — conditional_order_service.py

Purpose: Trigger conditional orders (stop, limit, trailing) when price conditions are met

The BrokerPriceMonitor checks prices for active conditional orders. Time-sensitive — a delayed price could miss a stop-loss trigger.

_try_streaming_hub_price() │ ├── Step 1: UPH check (NEW — first priority) │ ├── uph.get_quote(self.symbol) │ ├── if freshness in ('fresh', 'aging') and last > 0: │ │ └── return price ✓ │ └── else: fall through │ └── Step 2: Direct hub (existing — fallback) ├── self.data_hub.get_quote_price(self.symbol) └── return price or None

Pipeline 3: Streaming Price Monitor — conditional_orders/base.py

Purpose: Real-time price monitoring for individual conditional orders

Each conditional order (OCO, bracket, trailing stop) has its own StreamingPriceMonitor. The _query_hub() method is called on each monitoring cycle.

_query_hub() │ ├── Step 1: UPH check (NEW — first priority) │ ├── uph.get_quote(self.symbol) │ ├── if freshness in ('fresh', 'aging') and last > 0: │ │ └── return float(price) ✓ │ └── else: fall through │ └── Step 2: Direct hub (existing — fallback) ├── self.data_hub.get_quote_price(self.symbol) └── return float(price) or None

Integration Pattern — All Three Pipelines

Every integration follows the same defensive pattern:
  1. Try-import get_unified_price_hub (import guard — no crash if file missing)
  2. Check uph._shadow_mode == False (safety — does nothing if reverted to shadow)
  3. Check uph._poll_running (safety — does nothing if polling hasn't started)
  4. Call uph.get_quote(symbol)
  5. Check freshness in ('fresh', 'aging') (only use if within 5s of last price change)
  6. On any exception: pass (never let UPH crash existing pipelines)

Only one UPH instance ever exists. All consumers share the same cache. Access via get_unified_price_hub() module-level function.

Symbol Resolution System

_to_canonical(symbol)

Maps any symbol to its canonical form. Uses _INDEX_TO_CANONICAL map: SPXW→SPX, NDXP→NDX, VIXW→VIX, RUTW→RUT, DJXW→DJX. Non-index symbols pass through unchanged.

_symbol_for_hub(symbol, hub_key)

Reverse mapping: given a canonical symbol, what does this specific hub call it? E.g., Schwab uses SPXW for SPX, NDXP for NDX. Webull uses SPX directly.

// Example: User has SPX options position register_monitored_symbols(['SPX', 'SPXW']) → both canonicalize to 'SPX' → _monitored_symbols = {'SPX'} → proactive subscribe: Schwab: request_subscribe_equities({'SPXW'}) // Schwab uses SPXW IBKR: subscribe_symbol('SPX') // IBKR uses SPX
