Executive Summary

Objective

Build an integrated public filing tracker that monitors portfolio changes from U.S. Congress members, corporate insiders (CEO, CFO, board members), and institutional investors (hedge funds, pension funds, sovereign wealth funds). Surface actionable trade signals based on filing patterns, conviction scoring, and historical performance attribution.

Data Source Tiers

Key Capabilities

  • Real-time congressional trade alerts (House + Senate)
  • Insider transaction tracking with sentiment scoring (MSPR)
  • Quarterly 13F portfolio diffing for whale investors
  • Conviction scoring & historical win-rate per filer
  • Automated signal generation through existing pipeline
  • Web GUI dashboard with filtering and drill-down

Design Principles

  • Free-first: Finnhub (free) + SEC EDGAR (free) as primary sources
  • Layered enrichment: Add FMP/Quiver only when needed
  • Existing integration: Uses current bot infrastructure (DB, signals, GUI)
  • Rate-limit safe: Scheduled polling with exponential backoff
  • Compliance-aware: Respects SEC User-Agent policy & data usage terms
  • Auditable: Every filing stored with source link and raw JSON

Deduplication Strategy

Multiple sources may report the same filing. Dedup uses a composite hash key:

# Congressional trades
dedup_key = hash(f"{filer_name}|{symbol}|{transaction_date}|{amount}|{type}")

# Insider transactions (Form 4)
dedup_key = hash(f"{filer_name}|{symbol}|{transaction_date}|{shares}|{price}|{code}")

# 13F holdings (per quarter)
dedup_key = hash(f"{cik}|{cusip}|{quarter}|{shares}")

On collision, prefer the source with richer metadata (FMP > Finnhub > EDGAR for congressional; Finnhub > EDGAR for insider).

CREATE TABLE IF NOT EXISTS filing_trades (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    dedup_hash      TEXT UNIQUE NOT NULL,        -- Composite hash for dedup
    filing_type     TEXT NOT NULL,                -- 'CONGRESS', 'INSIDER', '13D'
    source          TEXT NOT NULL,                -- 'finnhub', 'fmp', 'edgar'

    -- Filer info
    filer_name      TEXT NOT NULL,                -- 'Nancy Pelosi', 'Tim Cook'
    filer_title     TEXT,                         -- 'Representative', 'CEO', 'Director'
    filer_party     TEXT,                         -- 'D', 'R', NULL (insiders)
    filer_state     TEXT,                         -- 'CA', 'OK', NULL
    filer_chamber   TEXT,                         -- 'house', 'senate', NULL

    -- Trade info
    symbol          TEXT NOT NULL,
    asset_type      TEXT,                         -- 'Stock', 'Option', 'ETF'
    transaction_type TEXT NOT NULL,               -- 'Purchase', 'Sale', 'Exchange'
    transaction_date TEXT NOT NULL,               -- YYYY-MM-DD
    disclosure_date TEXT NOT NULL,                -- YYYY-MM-DD
    shares          REAL,                         -- Exact share count (Form 4)
    price           REAL,                         -- Exact price (Form 4, often NULL for congress)
    amount_range    TEXT,                         -- '$15,001 - $50,000' (congress)
    amount_low      REAL,                         -- Parsed lower bound
    amount_high     REAL,                         -- Parsed upper bound
    owner           TEXT,                         -- 'Self', 'Spouse', 'Child'

    -- Scoring
    conviction_score REAL DEFAULT 0,              -- 0-100 composite score
    mspr            REAL,                         -- Insider sentiment (-100 to +100)
    disclosure_lag  INTEGER,                     -- Days between trade and disclosure
    cluster_id      TEXT,                         -- Group ID for clustered trades

    -- Metadata
    filing_url      TEXT,                         -- Link to original SEC/Senate/House filing
    raw_json        TEXT,                         -- Full API response preserved
    signal_sent     INTEGER DEFAULT 0,            -- 1 if alert was generated
    created_at      TEXT DEFAULT CURRENT_TIMESTAMP,
    updated_at      TEXT DEFAULT CURRENT_TIMESTAMP
);

Unified FilingTrade Dataclass

from dataclasses import dataclass, field
from typing import Optional
from datetime import date

@dataclass
class FilingTrade:
    filing_type: str               # 'CONGRESS', 'INSIDER', '13D'
    source: str                    # 'finnhub', 'fmp', 'edgar'
    filer_name: str
    symbol: str
    transaction_type: str         # 'Purchase', 'Sale'
    transaction_date: date
    disclosure_date: date

    # Optional enrichment
    filer_title: Optional[str] = None
    filer_party: Optional[str] = None
    filer_state: Optional[str] = None
    filer_chamber: Optional[str] = None
    asset_type: Optional[str] = None
    shares: Optional[float] = None
    price: Optional[float] = None
    amount_range: Optional[str] = None
    amount_low: Optional[float] = None
    amount_high: Optional[float] = None
    owner: Optional[str] = None
    mspr: Optional[float] = None
    filing_url: Optional[str] = None
    raw_json: Optional[str] = None

    @property
    def dedup_hash(self) -> str:
        import hashlib
        key = f"{self.filer_name}|{self.symbol}|{self.transaction_date}|" \
              f"{self.shares or self.amount_range}|{self.transaction_type}"
        return hashlib.sha256(key.encode()).hexdigest()[:16]

    @property
    def disclosure_lag_days(self) -> int:
        return (self.disclosure_date - self.transaction_date).days

Signal Generation Engine

Conviction Scoring Algorithm

Each filing is scored 0-100 based on multiple weighted factors. Scores above the user-configurable threshold (default: 65) generate an alert.

Factor Weight Logic Max Points
Trade Size 25% Larger amounts = higher conviction. $1M+ = max score. For congress: parse amount range midpoint. 25
Filer Track Record 25% Historical win rate (30-day forward return > 0). Pelosi 65%+ win rate = high. New filer = neutral (12.5). 25
Cluster Detection 20% Multiple filers buying same symbol within 14-day window. 3+ filers = max. Indicates institutional consensus. 20
MSPR Alignment 15% For insiders: MSPR > 50 on a purchase = strong. MSPR < -50 on a sale = strong. N/A for congress. 15
Disclosure Timing 10% Faster disclosure (< 7 days) = urgency signal. Congress max is 45 days; < 10 days is unusual. 10
Committee Relevance 5% Politician on relevant committee (e.g. Finance committee member trading bank stocks). Requires enrichment. 5

Signal Types Generated

🔴 High-Conviction Alert (Score ≥ 75)

  • Push notification to GUI dashboard
  • Discord webhook via existing /api/discord/send-signal-multi
  • Optional: auto-queue as trade signal for review

🔵 Standard Alert (Score 50-74)

  • Display in Filing Tracker tab
  • Optional Discord notification
  • Added to watchlist for monitoring

Cluster Alert

When 3+ distinct filers (politicians or insiders) buy the same symbol within a 14-day window, generate a cluster alert regardless of individual conviction scores. This is the strongest signal type historically.

# Cluster detection query
SELECT symbol, COUNT(DISTINCT filer_name) as filer_count,
       GROUP_CONCAT(DISTINCT filer_name) as filers,
       MIN(transaction_date) as first_trade,
       MAX(transaction_date) as last_trade
FROM filing_trades
WHERE transaction_type = 'Purchase'
  AND transaction_date >= date('now', '-14 days')
  AND signal_sent = 0
GROUP BY symbol
HAVING filer_count >= 3

13F Portfolio Diffing

Quarterly 13F analysis compares consecutive snapshots to identify:

  • New Positions: Symbol appears in current quarter but not previous
  • Increased Positions: Share count increased by >25%
  • Decreased Positions: Share count decreased by >25%
  • Exited Positions: Symbol in previous quarter but absent from current
# Diff two quarters
SELECT
    COALESCE(curr.symbol, prev.symbol) as symbol,
    prev.shares as prev_shares,
    curr.shares as curr_shares,
    CASE
        WHEN prev.shares IS NULL THEN 'NEW'
        WHEN curr.shares IS NULL THEN 'EXIT'
        WHEN curr.shares > prev.shares * 1.25 THEN 'INCREASE'
        WHEN curr.shares < prev.shares * 0.75 THEN 'DECREASE'
        ELSE 'HOLD'
    END as action
FROM filing_13f_snapshots curr
FULL OUTER JOIN filing_13f_snapshots prev
    ON curr.cik = prev.cik AND curr.cusip = prev.cusip
WHERE curr.quarter = '2026-Q1' AND prev.quarter = '2025-Q4'
    AND curr.cik = '{target_cik}'

Risk & Compliance

⚠️ Legal Considerations

  • SEC EDGAR: Free for any use. Must include User-Agent with contact info. No redistribution restrictions.
  • Senate EFD: Terms prohibit "commercial use" of data scraped directly. Third-party APIs (FMP, Finnhub) that aggregate this data are fine.
  • Finnhub: Free tier allows commercial use. Attribution not required but recommended.
  • FMP: API data can be used in applications. Check terms for redistribution limits.
  • Not financial advice: Filing data has disclosure lag (2-45 days). Prices may have moved significantly. Always display disclosure date prominently.

🛡️ Technical Risks & Mitigations

  • API downtime: Multi-source fallback (Finnhub fails → FMP → EDGAR). Circuit breaker pattern with backoff.
  • Rate limiting: Token bucket per API. Finnhub: 60/min, FMP: 250/day, EDGAR: 10/sec. Scheduler enforces.
  • Data staleness: Congressional filings have up to 45-day lag. Always show disclosure_lag_days in UI. Don't auto-trade stale filings (>7 days old).
  • Duplicate filings: Hash-based dedup across all sources. Prefer richer source on collision.
  • Schema changes: API responses validated against expected fields. Log and skip malformed records.
  • Amount ambiguity: Congressional amounts are ranges ($1K-$15K). Parse both bounds. Score uses midpoint.
