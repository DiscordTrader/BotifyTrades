03 — Encryption Architecture

Encryption at Every Layer

Credentials and sensitive data are protected using industry-standard symmetric encryption, with keys derived from license-bound or OS-level sources.

At-Rest Encryption

All broker credentials encrypted using Fernet symmetric encryption (AES-128 CBC mode with HMAC-SHA256 authentication) before database storage

🌐

In-Transit Encryption

All broker API communication over TLS 1.2+ with verified CA certificates via certifi bundle. WebSocket connections use WSS protocol

🗝

Key Management

Encryption keys derived from license activation binding or OS-level key file. Keys never transmitted externally. Supports environment variable override

Speaker Notes
Encryption uses the Python cryptography library's Fernet implementation (AES-128-CBC + HMAC-SHA256). Encryption keys can be derived from the user's license key via SHA-256 for portability across machines. Plaintext credentials exist only in memory during active API calls and are never written to disk or logs.

06 — Network Security

Network Security Controls

All external communication is secured with TLS encryption. The platform exposes no inbound network services to the internet.

External Network — TLS 1.2+ Encrypted
Application Layer — Localhost Only
Encrypted
Data Store
🔒

TLS Certificate Verification

All HTTPS and WSS connections validate server certificates against a trusted CA bundle (certifi). Prevents man-in-the-middle attacks on broker API communication.

🛡

No Inbound Exposure

The platform does not open any ports to the internet. The web dashboard is bound to 127.0.0.1 (localhost) and is accessible only from the local machine.

🔄

WebSocket Security

Real-time data streams from brokers and signal sources use WSS (WebSocket Secure) protocol with automatic reconnection and heartbeat monitoring.

⏱

Rate Limiting

Built-in rate limit management for all broker APIs prevents throttling and ensures compliance with broker-imposed API usage limits.

Speaker Notes
The platform's network footprint is minimal by design. Outbound connections go to: (1) broker REST APIs over HTTPS, (2) broker streaming APIs over WSS, (3) signal source WebSocket gateways over WSS. The only local service is the Flask web dashboard bound to 127.0.0.1:5000, inaccessible from outside the machine.

08 — Privacy & Data Retention

Privacy & Data Retention

The platform follows a data minimization approach — collecting only what is necessary for trading execution and storing it exclusively on the user's machine.

🛡 Privacy Principles

Data Minimization

Only data required for trade execution is processed. No browsing history, personal demographics, or behavioral analytics are collected.

Local-First Architecture

100% of user data resides on the local machine. No cloud databases, no third-party storage, no data replication.

User Data Sovereignty

The user has complete control over their data. Uninstalling the application removes all data. No residual data remains on external systems.
