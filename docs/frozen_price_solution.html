Problem 1: Duplicated Frozen Detection

Each pipeline independently checks if prices are stuck. Thresholds don't match. If Risk Engine detects frozen at 3s, StreamingPriceMonitor doesn't know. BrokerPriceMonitor has NO detection at all.

Risk Engine: 3s regular / 15s extended StreamingMonitor: 3s (market hours only) BrokerMonitor: ❌ NONE — blind polling
Impact: BrokerPriceMonitor can trade on a stale price forever

Problem 2: No Freshness Validation

StreamingPriceMonitor's _query_hub() and BrokerPriceMonitor's _try_streaming_hub_price() accept ANY price > 0 from hub cache — no age check. A 5-minute-old cached price passes as "fresh".

# base.py _query_hub() — NO freshness check price = self.data_hub.get_quote_price(self.symbol) if price and price > 0: # age = 300s? Still accepted! return float(price) # conditional_order_service.py — NO freshness check price = hub.get_quote_price(self.symbol) if price and price > 0: # ANY cached price accepted return price
Impact: Both conditional order pipelines can execute orders on stale prices

Problem 3: Cross-Broker Is Not Cross-Broker

Risk Engine's _try_cross_hub_price() requires is_streaming() AND _is_hub_live() AND max_age=2s — very strict. But if a symbol isn't subscribed on other hubs (like FCHL), all cross-hub checks return None. No one proactively subscribes symbols on secondary brokers.

# Risk Engine: strict but blind hub.is_streaming() # Hub must be streaming AND _is_hub_live(hub) # Quote within 120s AND max_age=2 # THIS symbol within 2s # Result for FCHL: all hubs return None # because FCHL isn't subscribed on any hub
Impact: FCHL flash crash — SL triggered at -16.83% instead of -10%

Problem 4: Schwab Subscription Delay

Schwab WebSocket drains pending subscriptions every 10 seconds. When Risk Engine calls request_subscribe_equities(), the actual WebSocket subscribe happens on the NEXT drain cycle — up to 10s later.

# schwab_streaming_client.py line 409 if now - self._last_pending_drain >= 10: pending = self._hub.drain_pending_subscriptions() await self.subscribe_equities(list(pending)) # Timeline: request → wait 0-10s → drain → subscribe # Worst case: 10s before Schwab even SENDS the subscribe
Impact: Cross-broker subscription can take 10-15s to produce first quote

Problem 5: REST Fallback Races Rate Limits

When streaming fails, all three pipelines independently call REST APIs. Risk Engine has MAX_REST_REPAIRS_PER_CYCLE=3, but conditional orders have no centralized rate coordination. Multiple symbols frozen simultaneously = rate limit exhaustion.

Webull: 30 req/60s = 1 every 2s Schwab: 120 req/60s = 1 every 0.5s Tastytrade: 120 req/60s = 1 every 0.5s IBKR: 50 req/1s = generous # 3 pipelines × N frozen symbols = uncoordinated burst
Impact: 429 rate limits during the exact moment you need price data most

Problem 6: UPH Exists But Is Shadow-Only

UnifiedPriceHub already subscribes to ALL hub events, caches prices, tracks freshness, emits events. But _shadow_mode=True means NO pipeline reads from it. It has 90% of the solution built, sitting unused.

# unified_price_hub.py line 114 self._shadow_mode = True # UPH already does: ✓ Subscribe to all 5 hubs (Webull/Schwab/IBKR/Tastytrade/T212) ✓ Cache with freshness classification (3s/5s/10s/30s) ✓ Cross-hub fill from all streaming hubs ✓ Event emission (quote_updated) ✗ NOT in read path of any pipeline ✗ NO frozen detection ✗ NO REST fallback
Impact: Perfectly good infrastructure gathering dust while pipelines reinvent the wheel

Layer 2: Frozen Detection Engine (NEW — add to UPH)

UPH already tracks last_changed_ts per symbol. Add a background sweep every 1s that flags symbols where price hasn't moved beyond threshold.

# New in unified_price_hub.py FROZEN_THRESHOLD = 2.0 # Regular hours: flag frozen after 2s FROZEN_EXTENDED = 15.0 # Extended hours: 15s (low volume normal) PROBE_COOLDOWN = 2.0 # Min time between cross-broker probes for same symbol def _frozen_sweep(self): """Runs every 1s. Checks all monitored symbols for frozen prices.""" now = time.time() threshold = FROZEN_THRESHOLD if self._is_regular_hours() else FROZEN_EXTENDED for sym, quote in self._cache.items(): if sym not in self._monitored_symbols: continue if quote.price_age_seconds >= threshold: if sym not in self._frozen_symbols: self._frozen_symbols[sym] = now self._emit('symbol_frozen', {'symbol': sym, 'age': quote.price_age_seconds}) self._trigger_cross_broker_probe(sym) # immediate action else: if sym in self._frozen_symbols: del self._frozen_symbols[sym] self._emit('symbol_recovered', {'symbol': sym})

Why 2s Not 3s?

During regular market hours, actively traded stocks update multiple times per second. 2s of no change is already unusual. By detecting at 2s, we have time to probe cross-broker AND initiate REST before the 5s staleness gate blocks SL evaluation in Risk Engine.

Why Sweep, Not Per-Tick?

Per-tick detection only fires when a tick arrives. If streaming dies silently (no ticks at all), per-tick never runs. The 1s sweep catches BOTH cases: same-price ticks AND no-ticks-at-all.

Layer 3: Cross-Broker Probe (NEW — add to UPH)

When a symbol is flagged frozen, UPH immediately checks ALL other hubs for a fresh price. If found, the cache updates and all consumers see the new price on their next read.

def _trigger_cross_broker_probe(self, symbol): """Check all hubs for a fresh price when primary source is frozen.""" canonical = self._to_canonical(symbol) current = self._cache.get(canonical) primary_hub = current.source_hub if current else None for hub_key, hub in self._get_hubs().items(): if hub_key == primary_hub: continue # skip the frozen source if not (hub.is_streaming() and self._is_hub_live(hub)): continue lookup = self._symbol_for_hub(symbol, hub_key) price = self._get_hub_price_if_fresh(hub, lookup, max_age=3.0) if price and price > 0: self._update_cache(canonical, {'last': price}, hub_key) self._emit('cross_broker_fill', {'symbol': canonical, 'source': hub_key}) return # first fresh cross-broker hit wins # No cross-broker had it → schedule REST probe self._schedule_rest_probe(symbol)

Layer 4: Smart REST Fallback (NEW — add to UPH)

When cross-broker hubs also don't have a fresh price, UPH makes a coordinated REST call. Rate-limit-aware, with broker preference ordering based on available budget.

def _schedule_rest_probe(self, symbol): """Rate-limit-aware REST fallback. Called when all hubs fail.""" from src.services.rate_limit_manager import RateLimitManager rlm = RateLimitManager.instance() # Priority: broker with most remaining budget → least disruptive broker_order = self._rank_brokers_by_budget(rlm) for broker_key in broker_order: if not rlm.can_call(broker_key): continue price = self._rest_quote(symbol, broker_key) if price and price > 0: rlm.record_call(broker_key) self._update_cache(symbol, {'last': price}, f'rest/{broker_key}') self._emit('rest_fill', {'symbol': symbol, 'broker': broker_key}) return # All brokers rate-limited — backoff 2s, re-probe on next sweep self._rest_backoff[symbol] = time.time() + 2.0 def _rank_brokers_by_budget(self, rlm): """Rank brokers by remaining API budget (highest first).""" scores = [] for key in ['schwab', 'ibkr', 'webull', 'tastytrade', 'alpaca']: remaining = rlm.remaining_budget(key) scores.append((remaining, key)) scores.sort(reverse=True) return [k for _, k in scores]

Layer 5: Proactive Subscription Manager (NEW — add to UPH)

When a symbol is registered for monitoring, UPH proactively subscribes it on ALL available streaming hubs — not just the primary broker. This ensures cross-broker data exists BEFORE a freeze happens.

def register_monitored_symbols(self, symbols, primary_broker=None): """Register symbols for monitoring. Subscribe on all available hubs.""" for sym in symbols: self._monitored_symbols.add(sym.upper()) # Proactive: subscribe on ALL streaming hubs, not just primary for hub_key, hub in self._get_hubs().items(): if not hub.is_streaming(): continue for sym in symbols: hub_sym = self._symbol_for_hub(sym, hub_key) if hub_key == 'schwab' and hasattr(hub, 'request_subscribe_equities'): hub.request_subscribe_equities({hub_sym}) elif hub_key == 'webull' and hasattr(hub, 'subscribe_symbol'): try: hub.subscribe_symbol(hub_sym) except: pass # IBKR + Tastytrade subscribe via their own hub methods

The single biggest fix for the FCHL scenario isn't faster detection — it's having data to detect WITH. Currently, FCHL was never subscribed on any cross-broker hub. The proactive subscription ensures that by the time a freeze happens, at least 2-3 hubs have the symbol subscribed and potentially streaming.

Why This Solution Is Safe

No Code Removed

All existing frozen detection, cross-broker, and REST logic stays intact. UPH is added as a first-check layer. If UPH fails or returns stale data, the existing pipeline logic handles it exactly as before.

Incremental Rollout

Each change is independently deployable and revertible. Start with Schwab drain. Then UPH frozen detection in observation mode. Then gradually enable REST and subscription features.

Works for Both Stocks & Options

UPH already has _HUB_SYMBOL_MAP for index mapping. Option quotes use raw_symbol keys. The existing canonical normalization handles SPX↔SPXW, NDX↔NDXP cross-broker differences.

Rate Limits Protected

Smart REST uses RateLimitManager to check budget before each call. Picks the broker with most remaining capacity. Never blind-fires REST calls. Falls back to 2s backoff if all brokers rate-limited.
