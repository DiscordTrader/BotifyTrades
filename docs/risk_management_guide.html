⚡ Exit Evaluation Priority

When the risk engine evaluates a position, it checks exit conditions in this exact order. The first condition that triggers wins.

1. Hard Stop Loss
Checks the fixed stop_loss_price on the cache. If price ≤ stop price, exits immediately. This is the raw price-based stop.
2. Channel Stop Loss
Evaluates the per-channel SL using priority chain: Manual Override → Dynamic SL → Channel SL%. If the position is down more than the SL%, sells everything.
3. Tiered Profit Targets (T1/T2/T3/T4)
Checks each tier sequentially. If price is up ≥ tier%, sells that tier's quantity. Only one tier fires per evaluation cycle.
If escalation_only_mode is on: tiers are marked as hit but no sell is placed — SL escalation handles the exit.
4. EMA Candlestick Exit
If EMA risk is enabled, checks for bearish EMA cross or consecutive no-trend candles. Can trigger a full exit.
5. Enhanced Risk Engine
Evaluates (in this sub-order): Dynamic SL trigger → Giveback Guard → Early Trailing exit. First trigger wins.
6. Trailing Stop
If trailing stop is enabled and activated (price reached activation %), checks if current price dropped below the trailing stop price.
7. Global Risk (fallback)
Only evaluated if no channel settings are configured. Uses global risk settings as a safety net.
ℹ
Stale Price Protection: If the price hasn't changed for an extended period, the system blocks stop-loss exits (to avoid acting on stale data) but still allows profit-target exits (selling at a stale high is favorable).

🎯 Profit Targets (T1 / T2 / T3 / T4)

Sell portions of your position as price rises through up to 4 profit levels.

Settings

SettingTypeDefaultDescription
profit_target_1_pctfloat0.0% gain to trigger Tier 1 sell
profit_target_2_pctfloat0.0% gain to trigger Tier 2 sell
profit_target_3_pctfloat0.0% gain to trigger Tier 3 sell
profit_target_4_pctfloat0.0% gain to trigger Tier 4 sell

Set to 0 to disable a tier. Tiers must be in ascending order (T1 < T2 < T3 < T4).

How Many Shares Sell at Each Tier?

Three methods are checked in this priority order. The first one that has a value wins:

PriorityMethodSettingHow It Works
1st Fixed Quantity profit_target_qty_1..4 Sell exactly this many shares. Capped to max sellable (after reserving runner shares).
2nd Trim Percentage profit_target_trim_pct_1..4 Sell this % of the sellable shares. Uses floor() rounding. E.g., 30% of 75 sellable = 22 shares.
3rd Auto-Calculate none (default) Splits sellable shares equally across remaining active tiers.

Default for all qty/trim settings is None (not set), which means auto-calculate kicks in.

How Tiers Execute

Tiers fire sequentially. T2 only evaluates after T1 has filled. T3 after T2, etc.

T1 hit → Sell T1 qty → Mark tier1_hit = true → Wait for fill ↓ T2 hit → Sell T2 qty → Mark tier2_hit = true → Wait for fill ↓ T3 hit → Sell T3 qty → Mark tier3_hit = true → Wait for fill ↓ T4 hit → Sell T4 qty → Mark tier4_hit = true → Done

A tier is skipped if:

  • Its tierN_hit flag is already true
  • There is a pending (unfilled) order for that tier
  • The tier % is set to 0 (disabled)

💰 Trim Order Mode

Controls HOW profit target sells are executed at the broker.

Market Mode

trim_order_mode'market'
  • The software risk engine detects PT hit, sends a market sell
  • Fills immediately at the current bid price
  • No limit order pre-placed at broker
  • No native OCO — only a standalone stop order is placed
✓
Best for: volatile stocks, fast fills, guaranteed execution

Limit Mode

trim_order_mode'limit'
  • A limit sell order is pre-placed at the broker at the PT price (minus an offset)
  • Can use native OCO brackets that link SL and PT
  • Broker fills automatically when price reaches the limit
  • May not fill in fast-moving markets
ℹ
Best for: liquid stocks, better fill prices, broker-managed protection

Limit Offset Settings

When trim_order_mode = 'limit', these control how far below the current price the limit sell is set:

SettingTypeDefaultDescription
trim_limit_offset_modestring'dollar''dollar' or 'percent'
trim_limit_offsetfloat0.01Dollar amount below current price (used when mode = dollar)
trim_limit_offset_pctfloat2.0Percent below current price (used when mode = percent)

Dollar mode (stocks ≥ $1.00): Rounds to nearest nickel bracket

Current price: $50.00, offset: $0.05 Base = $50.00 - $0.05 = $49.95 Cents digit = 5 (≥ 5), so limit = floor($49.9) + $0.09 = $49.99

Dollar mode (penny stocks < $1.00): Uses 4 decimal precision

Current price: $0.4500, offset: $0.01 Limit = $0.4500 - $0.01 = $0.4400 If result ≤ 0: fallback to current_price * 0.92 (8% haircut)

Percent mode:

Current price: $50.00, offset_pct: 2% Limit = $50.00 * (1 - 2/100) = $49.00

When Is Native OCO Used?

Native OCO (Schwab orderStrategyType: "OCO") links a stop-loss and a profit-target into one order. When one leg fills, the other auto-cancels.

All of these must be true:

#ConditionWhy
1trim_order_mode = 'limit'Market mode has the risk engine handle PT sells directly
2SL price > 0OCO needs a valid stop-loss leg
3PT price > 0OCO needs a valid profit-target leg
4PT quantity > 0Must have shares to sell at PT
5Position is equity (not an option)OCO is only supported for stocks
6broker_bracket_mode allows both SL and PTBoth legs must be permitted
⚠
If any condition fails, the system falls back to separate stop and limit orders. If the OCO API call fails (e.g., 429 rate limit), it also falls back to separate orders.

OCO + Standalone Stop Pattern

When the position has more shares than the T1 trim quantity, the system places two orders:

Example: 100 shares, T1 sells 25, SL at $45, PT at $55 OCO Order (25 shares): Leg 1: STOP SELL 25 @ $45.00 (SL for trim shares) Leg 2: LIMIT SELL 25 @ $55.00 (PT for trim shares) → If PT fills: SL auto-cancels for these 25 shares → If SL triggers: PT auto-cancels for these 25 shares Standalone STOP (75 shares): STOP SELL 75 @ $45.00 (SL for remaining shares) → Always active, independent of OCO When SL triggers: Both OCO-SL (25) + Standalone (75) fire = 100 share exit When PT fills: OCO-SL auto-cancels. Standalone STOP (75) stays active.

Cascade After a Tier Fills

When a tier's order fills, the system automatically sets up brackets for the next tier:

Step 1: Cancel old brackets
Cancel the filled tier's OCO (or separate PT order). Cancel standalone stop.
Step 2: Calculate next tier
Get the next tier's PT price and quantity. If trim_order_mode = 'market', skip PT placement — risk engine will handle.
Step 3: Escalate SL
If Dynamic SL is enabled, calculate the new (higher) SL price based on the profile.
Step 4: Place new brackets
Place new OCO (or separate orders) for next tier's qty + updated SL. Place new standalone stop for remaining shares.

SL Order Settings

SettingTypeDefaultDescription
sl_order_modestring'limit''market' (guaranteed fill) or 'limit' (better price, may not fill)
sl_limit_offsetfloat0.03Offset % below SL price for limit orders

OCO Duplicate Sell Prevention

When OCO is managing a tier's PT at the broker, the software risk engine suppresses its own PT sell for that same tier. This prevents both the OCO limit and the risk engine from selling the same shares.

Check in _evaluate_exit_conditions: If broker_oco_order_id exists AND the tier being hit == broker_pt_tier (the tier OCO is managing) → Return no_exit() — let the OCO handle it at the broker

🛑 Stop Loss

Protects the position from the moment of entry. Sells everything if price drops too far.

Settings

SettingTypeDefaultDescription
stop_loss_pctfloat0.0% below entry to trigger stop loss. 0 = disabled.
Example: Entry = $50.00, SL = 10% SL price = $50.00 × (1 - 10/100) = $45.00 If current price ≤ $45.00 → SELL ALL shares

SL Source Priority Chain

Multiple systems can set the stop loss price. They are checked in this order — first match wins:

PrioritySourceSettingCondition
1st Manual Override (price) manual_sl_price exit_strategy_mode != 'risk' AND manual_sl_price is set
2nd Manual Override (%) manual_sl_pct exit_strategy_mode != 'risk' AND manual_sl_pct is set
3rd Dynamic SL dynamic_sl_price enable_dynamic_sl = true AND dynamic_sl_price is set
4th Channel SL stop_loss_pct Always (default fallback)
⚠
Dynamic SL floor rule: When Dynamic SL is active, if the dynamic SL price is looser (lower) than the channel SL floor, the channel SL takes over. This prevents Dynamic SL from offering less protection than the base SL. Specifically: if dynamic_sl_price > channel_sl_floor, channel SL is used instead.

📈 Dynamic SL (SL Escalation)

Automatically moves the stop loss upward as profit targets are hit. Locks in progressively more profit.

Settings

SettingTypeDefaultDescription
enable_dynamic_slboolfalseEnable/disable Dynamic SL
dynamic_sl_profilestring'standard''conservative', 'standard', or 'aggressive'

Profile Values

Each profile defines where the SL moves (relative to entry price) after each tier is hit:

After Tier HitConservativeStandardAggressive
T1 Breakeven (0%) Breakeven (0%) -2% (allow small loss)
T2 +3% +5% Breakeven (0%)
T3 +8% +10% +8%
T4 +15% +17% +15%
ℹ
SL only ratchets up, never down. Once the SL moves to +5%, it stays there even if a new calculation gives a lower value.
Example: Standard Profile Walkthrough
Entry: $10.00 | Initial SL: $9.00 (-10%) Price hits T1 (7.8% = $10.78): → Sell T1 shares → Dynamic SL = $10.00 × (1 + 0/100) = $10.00 (breakeven) → Broker stop updated: $9.00 → $10.00 Price hits T2 (10% = $11.00): → Sell T2 shares → Dynamic SL = $10.00 × (1 + 5/100) = $10.50 (+5%) → Broker stop updated: $10.00 → $10.50 Price hits T3 (15% = $11.50): → Sell T3 shares → Dynamic SL = $10.00 × (1 + 10/100) = $11.00 (+10%) → Broker stop updated: $10.50 → $11.00

Ratchet Beyond Last Tier

After all configured tiers are hit and price keeps rising, the SL ratchets up to maintain the same "giveback distance" as the last tier:

Formula (after all tiers hit, price above highest tier): giveback_pct = highest_tier_pct - profile_sl_pct_for_that_tier (minimum 5% if calculated value < 1%) If current_pnl > highest_tier_pct: ratchet_sl_pct = current_pnl - giveback_pct ratchet_sl_price = entry × (1 + ratchet_sl_pct / 100) Safety cap: SL is always at least 2% below current price Example (Standard profile, T1=8%, T2=15%, T3=25%): highest_tier = T3 at 25%, profile T3 SL = +10% giveback = 25% - 10% = 15% Price at +30%: SL = 30% - 15% = +15% above entry Price at +40%: SL = 40% - 15% = +25% above entry → SL trails 15% behind the current profit, ratcheting up

Safety Cap

The Dynamic SL price is always capped at 2% below the current price. If the calculated SL would be at or above the current price:

If sl_price ≥ current_price: capped_sl = current_price × 0.98 If capped_sl > entry_price: use capped_sl Else: use entry_price (floor at breakeven)

Broker Sync

When the Dynamic SL price changes, the broker stop order is updated:

  • If OCO is active: Cancel old OCO → Place new OCO with new SL + same PT price
  • If standalone stop: Cancel old stop → Place new stop at the escalated SL price
  • The standalone stop quantity is total_qty - oco_qty (shares not covered by OCO)

🔒 Early Trailing

Locks in breakeven early, then ratchets up in fixed profit steps. Each step is permanent.

Settings

SettingTypeDefaultDescription
enable_early_trailingboolfalseEnable/disable Early Trailing
early_trailing_activation_pctfloat5.0% gain to activate and lock breakeven
early_trailing_step_pctfloat3.0Each step locks this much additional profit

How It Works

Inactive (below activation %)
Channel SL protects position. No breakeven lock yet.
Condition: pct_change < activation_pct
Activated: Breakeven Locked (Step 0)
early_stop_price = entry_price (breakeven)
early_steps_locked = 0
Trigger: pct_change ≥ activation_pct
Step N Locked
Formula: expected_steps = floor((pct_change - activation_pct) / step_pct)
If expected_steps > early_steps_locked:
  new_stop_pct = expected_steps × step_pct
  early_stop_price = entry_price × (1 + new_stop_pct / 100)
Steps only increase, never decrease.
Exit Triggered
current_price ≤ early_stop_price → Sell all shares
Locks in the profit from the last step.
Step-by-Step Example (activation=5%, step=3%)
Entry: $10.00 | Channel SL: $9.00 (-10%) +4.0% ($10.40): Inactive. SL at $9.00. +5.0% ($10.50): ACTIVATED → stop = $10.00 (breakeven), steps = 0 +7.9% ($10.79): No new step. Need +8% for step 1. Stop at $10.00 +8.0% ($10.80): STEP 1 → stop = $10.00 × 1.03 = $10.30, steps = 1 +10.9%($10.99): No new step. Need +11% for step 2. Stop at $10.30 +11.0%($11.10): STEP 2 → stop = $10.00 × 1.06 = $10.60, steps = 2 +14.0%($11.40): STEP 3 → stop = $10.00 × 1.09 = $10.90, steps = 3 Price drops to $10.90: EXIT → Sell all, locking +9% profit
⚠
Cannot combine with Trailing Stop. Early Trailing and Trailing Stop are mutually exclusive. Enabling both will cause an error. Choose one.

Interaction with Dynamic SL

Both Early Trailing and Dynamic SL can be active at the same time. The higher of the two stop prices is used as the effective SL. The broker stop is synced to whichever is higher.

Example after T1 hit: Dynamic SL says: $10.00 (breakeven via standard profile) Early Trailing says: $10.30 (step 1 locked at +3%) → Effective SL: $10.30 (early trailing wins — it's higher)

📍 Trailing Stop

A traditional trailing stop that activates at a profit threshold and trails a fixed % below the highest price.

Settings

SettingTypeDefaultDescription
trailing_stop_pctfloat0.0% distance below highest price. 0 = disabled.
trailing_activation_pctfloat15.0% gain required before trailing activates

How It Works

Phase 1: Before Activation

  • The system tracks highest_price (ratchets up, never down)
  • Channel SL protects the position
  • Trailing is dormant until price reaches +activation_pct%

Phase 2: Activated

  • Trailing stop price = highest_price × (1 - trailing_pct / 100)
  • Stop moves up as price rises, never moves down
  • If price drops to trailing stop → exit
Example (trailing=5%, activation=15%)
Entry: $10.00 $11.00 (+10%): Not activated yet (need +15%) $11.50 (+15%): ACTIVATED trailing = $11.50 × 0.95 = $10.925 $12.00 (+20%): trailing = $12.00 × 0.95 = $11.40 (ratcheted up) $13.00 (+30%): trailing = $13.00 × 0.95 = $12.35 (ratcheted up) $12.50 (-3.8%): trailing stays at $12.35 (never moves down) $12.35: EXIT → Sells at trailing stop price

Leave Runner Interaction

If Leave Runner is enabled, trailing stop sells everything except the runner shares:

100 shares, 25% runner Trailing triggers: → Sell 75 shares (trailing exit) → Keep 25 shares (runner)

💪 Giveback Guard

Prevents giving back too much unrealized profit. Exits when drawdown from peak profit exceeds the allowed amount.

Settings

SettingTypeDefaultDescription
enable_giveback_guardboolfalseEnable/disable Giveback Guard
giveback_allowed_pctfloat30.0Maximum % of peak profit that can be given back

When Does It Activate?

Giveback Guard activates (starts monitoring) when either condition is true:

  • Tier 2 has been hit (tier2_hit = true), OR
  • max_pnl_seen reaches the activation threshold:
    • If trailing stop configured: uses trailing_activation_pct
    • If early trailing configured: uses early_trailing_activation_pct
    • Otherwise: defaults to 30%

Exit Trigger Formula

Formula: giveback_threshold = max_pnl_seen × (1 - giveback_allowed_pct / 100) If current_pnl ≤ giveback_threshold → EXIT ALL Example: max_pnl_seen = 20% (position peaked at +20%) giveback_allowed_pct = 30% giveback_threshold = 20 × (1 - 30/100) = 20 × 0.70 = 14% If PnL drops to 14% or below → EXIT (Gave back 6 percentage points of the 20% peak = 30%)

🏃 Leave Runner

Reserves a portion of the position that profit target sells will never touch. Lets the runner capture extended moves.

Settings

SettingTypeDefaultDescription
leave_runner_enabledboolfalseEnable/disable Leave Runner
leave_runner_pctfloat25.0% of the position to keep as runner

How It Works

Formula: runner_qty = max(1, floor(current_qty × leave_runner_pct / 100)) max_sellable = current_qty - runner_qty Example: 100 shares, 25% runner runner_qty = max(1, floor(100 × 0.25)) = 25 shares reserved max_sellable = 100 - 25 = 75 shares available for PT sells All tier quantities are capped to max_sellable. After all tiers sell their shares, 25 runner shares remain.

📐 Escalation-Only Mode

Marks tier hits for SL escalation without placing any profit-target sells.

Settings

SettingTypeDefaultDescription
escalation_only_modeboolfalseWhen on, tiers are marked as hit but no partial sells fire

How It Works

When escalation_only_mode = true:

  • Price crossing a tier threshold marks tierN_hit = true
  • No sell order is placed for that tier
  • Dynamic SL escalates based on which tiers are hit (if enabled)
  • The SL ratchet protects profit — the exit comes from the SL, not from partial sells
  • Broker brackets: only SL is placed (no PT orders at broker, since pt1_qty = 0)
⚠
Escalation-only mode requires either enable_dynamic_sl or enable_early_trailing to work. Without one of these, tiers are marked but the SL never escalates, providing no benefit.
