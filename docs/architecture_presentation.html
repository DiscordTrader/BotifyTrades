Current State

Current Architecture

Each user runs an independent bot instance with direct broker connections for both price data and order execution.

CURRENT MODEL: Single-User Bot Instance Discord Signal ──> Signal Parser ──> Conditional Order Monitor | Price from User's Own Broker (Webull MQTT / Schwab WS / Tastytrade DXLink) | Condition Met ──> Execute on Broker | Risk Engine Monitors via Broker Stream (Position price from same broker connection) | Exit Condition ──> Close Position
⚡
UnifiedPriceHub
Aggregates prices from all connected broker data hubs into a single source of truth per bot instance.
⚙
Risk Engine
1-second monitoring cycle with SL, PT tiers (1-4), trailing stops, giveback guard, EMA exits, and early trailing.
🔗
Direct Broker SDK
Custom integrations for Webull, Schwab, Tastytrade, IBKR, Alpaca with per-broker streaming clients.
📊
Position Ledger
Tracks all positions with partial exits, P&L, risk state, and broker sync reconciliation every 30 seconds.
⚠ Key Limitation
Every user must independently authenticate and maintain their own broker streaming connections. Price data quality depends entirely on each user's broker credentials and network stability.

Data Source Analysis

Broker-Only vs Dedicated Market Data

Can broker connections alone provide real-time data to all users? Deep analysis of data source options.

✖ Broker-Only Data
Using Webull/Schwab/Tastytrade connections
✖ Session-dependent: token expire = all users lose data
✖ No SLA: brokers can change APIs without notice
✖ ~500 symbol limit per connection
✖ Redistributing data violates broker ToS
✖ Single point of failure per broker
✔ Free (no additional cost)
VS
✔ Polygon.io + Broker Backup
Primary: Polygon | Fallback: Schwab/Webull
✔ API key auth: no user sessions needed
✔ 99.9% SLA with contractual guarantees
✔ Unlimited symbols (all US equities + options)
✔ Licensed for redistribution (paid tier)
✔ Multi-source failover eliminates SPOF
● $199-499/mo (covered by ~15 subscribers)

▶ Verdict: Hybrid Approach Required

Broker-only data is too fragile for production SaaS and likely violates Terms of Service for redistribution. Use Polygon.io as primary ($199/mo) with existing broker streams as automated failover. The cost is trivially recovered from 10-20 user subscriptions.

Broker Integration

SnapTrade Integration Analysis

Unified broker API replacing direct SDK integrations for order execution across 20+ brokers.

What SnapTrade Provides

  • Unified API for 20+ brokers (Schwab, Webull, Tastytrade, IBKR, etc.)
  • OAuth2 broker connection flow (user connects via redirect)
  • Order placement API (market, limit, stop orders)
  • Position and account balance fetching
  • User credentials never touch your servers
  • Handles broker token refresh automatically

What SnapTrade Does NOT Provide

  • Real-time price streaming (REST only, polling)
  • Full bracket order support (limited)
  • Complex order modifications
  • Options chain depth for all brokers

🔒 Critical Confirmation

SnapTrade cannot provide real-time streaming prices — this confirms the Master Application with Polygon.io is mandatory for price data. SnapTrade handles execution only, while all price monitoring (conditional orders + risk engine) relies on the centralized price feed.

⚠ Critical Risk: User Bot Offline
In Model A, if a user closes their bot, the risk engine stops monitoring their open positions. This is the strongest argument for moving to Model B (backend risk engine) as soon as possible. Open positions without active risk monitoring can result in catastrophic losses.

Phased Roadmap

Progressive transformation from current single-user model to full multi-tenant SaaS.

▶ Final Recommendation

The transformation is fully achievable. Broker-only data is insufficient for production SaaS — you need Polygon.io (or equivalent) for reliable, licensed, low-latency market data redistribution. The centralized model actually reduces latency compared to current per-user broker streaming and eliminates authentication complexity.

Start with Phase 1 to validate the architecture with minimal risk. SnapTrade solves broker coverage (20+ vs current 5). The $199/mo Polygon cost is trivially covered by subscriber revenue. Move risk engine to backend in Phase 2 to solve the critical "bot offline" gap. Full SaaS in Phase 3 unlocks recurring revenue at scale.
